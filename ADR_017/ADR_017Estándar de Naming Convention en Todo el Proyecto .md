# ADR 017: Estándar de Naming Convention en Todo el Proyecto 
**Estado:** Aceptado

## Contexto
El equipo crece y necesita mantener reglas claras para nombrar archivos, carpetas, clases, endpoints y módulos.

## Decisión
Definir un documento oficial con las convenciones:
- camelCase → variables y funciones  
- PascalCase → clases y DTOs  
- kebab-case → nombres de archivos  
- snake_case → columnas en la base de datos  

## Alternativas Consideradas
1. Estilo libre  
2. Guías de Google  
3. Convenciones de Python  

## Consecuencias
- Código más consistente.  
- Mayor mantenibilidad.  
- Facilita colaboración entre equipo.
