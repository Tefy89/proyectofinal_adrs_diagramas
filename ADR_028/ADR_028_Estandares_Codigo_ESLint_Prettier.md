# ADR 028: Definición de Estándares de Código con ESLint y Prettier
**Estado:** Aceptado

## Contexto
El proyecto crece y varios desarrolladores contribuyen.  
Es necesario mantener un estilo consistente.

## Decisión
Utilizar **ESLint + Prettier** con reglas estrictas compartidas.

## Alternativas Consideradas
1. Estilo libre.  
2. Reglas personalizadas no unificadas.

## Consecuences
- Código legible.  
- Menos errores de formato.  
- Mejor mantenibilidad.
