# ADR 002: Selección de Angular para Frontend Web
Decisión técnica para una interfaz robusta, modular y escalable.  
**Estado:** Aceptado  

## Contexto
El frontend de MANAGIX debe ser estable, tipado, modular y preparado para crecer.  
Se requiere un framework con herramientas empresariales, arquitectura clara y buena integración con TypeScript.

## Decisión
Utilizar **Angular 16+** como framework principal del frontend administrativo.

## Alternativas Consideradas
- **React + TypeScript:** Flexible, pero requiere más decisiones arquitectónicas.
- **Vue.js:** Más simple, pero con menor orientación empresarial.
- **Svelte:** Muy moderno, pero con ecosistema menos maduro.

## Consecuencias
- Arquitectura definida y consistente.  
- Potente CLI y herramientas de build.  
- Organización ideal para aplicaciones complejas.  
- Curva de aprendizaje elevada.  
- Tamaño de bundle mayor comparado con alternativas.
